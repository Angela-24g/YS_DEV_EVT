# 어노테이션 문제 확인
```bash
Consider defining a bean of type 'com.example.domain.webauthn.mapper.CredentialMapper' in your configuration.
이렇게 빈 관련 문제가 있는 경우가 종종 발견된다. 이는, mapper나 mapperscan쪽 설정이 이상한다든지, 캐시가 쌓여서 프로젝트 자체로의 문제가 염려됨.

그리하여 다음과 같은 방법을 취하면 됨
@MapperScan(basePackages = {"com.example.domain.*.store"}, sqlSessionFactoryRef = "primarySqlSessionFactory", annotationClass = MapperPrimary.class)
basePackages에서, 난 경로가 "com.example.domain.*.store" 이게 맞았는데, 프로젝트를 여러 개 쓰다보니 "com.example.domain.*.repository"로 쓰게되어 매칭이 안되었던 것

캐시무효화는
파일 > 캐시무효화 > 그냥다시시작  누르면, 캐시 날아감

```
